# Literal Memo 用户指南

## 快速入门

Literal Memo 是一款受 [howm](https://kaoriya.github.io/howm/) 启发的极简笔记应用。

**核心理念**：写。丢。搜。

没有文件夹。没有标签。没有链接。只管把想法丢进堆里，需要的时候搜索即可。

## 基本操作

### 创建笔记

1. 点击 **+** 浮动按钮
2. 编写笔记（支持 Markdown）
3. 点击返回箭头或返回上一页即可保存

### 编辑笔记

1. 从列表中点击笔记——直接进入编辑模式
2. 修改内容
3. 点击返回箭头保存并同步

点击**浮动按钮**（眼睛图标）切换到预览模式。再次点击（铅笔图标）返回编辑模式。

### Markdown 工具栏

编辑时，键盘上方会出现工具栏，提供常用 Markdown 格式的快捷按钮（粗体、斜体、标题、链接等）。

### 删除笔记

**从列表中**：在笔记上向左滑动 → 确认对话框。

**从编辑界面**：点击**删除**按钮（右上角垃圾桶图标） → 确认。

删除的笔记会从应用中移除。如果开启了 Git 同步，它们会被移动到仓库的 `trash/` 目录——并非永久删除，只是移出视线。

## 搜索笔记

在列表底部的**搜索栏**中输入内容，然后点击搜索键。

- 搜索结果会显示内联片段，匹配文本会高亮
- 点击**返回**清除搜索，返回完整列表

## 分享链接

你可以从其他应用将 URL 或文本直接分享到 Literal Memo：

1. 在浏览器或任何应用中，点击**分享**
2. 选择 **Literal Memo**
3. 系统会创建一条新笔记，内容包含：
   - URL 在预览模式下变为可点击链接
   - 带标题的文本变为带标题的笔记

**示例**：分享一篇新闻文章会生成：
```markdown
# 文章标题

[文章标题](https://example.com/article)
```

这对"稍后阅读"工作流非常有用：随手分享链接，稍后在 PC 上阅读。

## Git 同步

支持 **GitHub** 同步。

### 设置

1. 在 GitHub 上创建一个**私有**仓库（例如 `username/literalmemo`）
2. 创建个人访问令牌：
   - 前往 GitHub → Settings → Developer settings → Personal access tokens
   - 生成一个具有 `repo` 权限的令牌
3. 在 Literal Memo 中，进入**设置**
4. 输入你的令牌和仓库（格式：`username/repo`）
5. 点击**同步**

### 同步机制

- **自动同步**：应用会自动同步：
  - 启动应用时
  - 返回应用时（切换到前台）
  - 保存或删除笔记后
- **本地优先**：你的编辑先保存在本地，然后同步
- **远程权威**：同步后，远程仓库拥有权威版本

### 切换仓库

如果你更改了主机或仓库，下次同步时会清除所有本地数据，然后从新仓库重新下载所有内容。执行此操作前会显示警告。

### 多设备使用

在多台设备上使用 Literal Memo 时：

1. **编辑前让应用先同步**：应用会在启动时自动执行
2. **不要同时在多台设备上编辑同一条笔记**
3. **编辑后同步**：应用会自动执行

### 冲突解决

如果你在同步前在多台设备上编辑了同一条笔记：

- 先同步的设备"胜出"
- 另一台设备会下载已同步的版本
- 第二台设备上未保存的更改将被覆盖

这是有意为之的设计。对于简单的笔记来说，这通常不是问题。如果需要恢复内容，所有更改都保存在 Git 历史中。

### 从 Git 恢复

如果不小心丢失了内容：

1. 前往你的仓库
2. 点击文件 → History
3. 找到包含你内容的提交
4. 按需复制或回退

### Neovim 集成

如果你使用 Neovim，可以直接访问笔记：
```lua
-- ~/.config/nvim/lua/custom/literalmemo.lua
local M = {}
local pile_path = vim.fn.expand('~/path/to/literalmemo/pile/')

M.new = function()
  local filename = os.date('!%Y%m%d_%H%M%S') .. '.md'
  vim.cmd('edit ' .. pile_path .. filename)
end

M.list = function()
  require('fzf-lua').files({ cwd = pile_path })
end

M.search = function()
  require('fzf-lua').live_grep({ cwd = pile_path })
end

vim.keymap.set('n', '<leader>mn', M.new, { desc = '新建笔记' })
vim.keymap.set('n', '<leader>ml', M.list, { desc = '列出笔记' })
vim.keymap.set('n', '<leader>ms', M.search, { desc = '搜索笔记' })

return M
```

## 故障排除

### 同步不工作？

- 检查网络连接
- 确认令牌具有 `repo` 权限
- 确保仓库存在且可访问
- 在设置中确认 Git 同步已连接

### 笔记没有出现在其他设备上？

- 启动后等待几秒，让同步完成
- 检查文件是否存在于你的仓库中
- 尝试在设置中手动触发同步

### 丢失了笔记？

删除的笔记会被移动到仓库的 `trash/` 目录。要恢复，直接在仓库中将文件从 `trash/` 移回 `pile/`，然后同步应用。

所有更改也保存在 Git 历史中，如有需要可以恢复旧版本内容。
